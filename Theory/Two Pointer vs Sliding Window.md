
# 🚀 투 포인터 vs 슬라이딩 윈도우

배열이나 리스트와 같은 선형 자료구조를 효율적으로 탐색하기 위한 두 기법은 모두 **시간 복잡도 $O(n)$** 을 목표로 하지만, 구체적인 전략에서 차이가 있습니다.

---

## 1. 투 포인터 (Two Pointers)
두 개의 인덱스(포인터)를 사용하여 조건에 맞는 **'지점'** 이나  **'쌍'** 을 찾는 기법입니다.

* **동작 방식**: 
    * **양 끝에서 안쪽으로**: 정렬된 배열에서 두 수의 합을 찾을 때 주로 사용.
    * **같은 방향에서 다른 속도로**: 연결 리스트의 중간 지점이나 사이클을 찾을 때 사용 (Fast/Slow Pointers).
* **핵심 특징**: 두 포인터 사이의 **간격이 일정하지 않고 유동적**입니다.
* **언제 쓰는가?**:
    * 정렬된 배열에서 특정 합(Target Sum)을 만족하는 쌍을 찾을 때.
    * 배열 안에서 데이터의 위치를 서로 바꿔야 할 때 (In-place swap).
    * 중복된 요소를 제거하거나 특정 값만 뒤로 보낼 때.



---

## 2. 슬라이딩 윈도우 (Sliding Window)
고정되거나 가변적인 크기의  **'창문(Window)'** 을 옆으로 밀면서 그 안의 데이터를 처리하는 기법입니다.

* **동작 방식**: 시작 포인터와 끝 포인터가 **항상 같은 방향**으로 이동하며, 창문 내부의 데이터 상태를 유지합니다.
* **핵심 특징**: 윈도우가 한 칸 이동할 때 **"새로 들어오는 값은 더하고, 나가는 값은 빼는"** 방식으로 중복 계산을 방지합니다.
* **언제 쓰는가?**:
    * **"연속된(Consecutive)"** 데이터의 합이나 평균을 구할 때.
    * **"부분 문자열(Substring)"** 내의 문자 빈도수를 확인할 때.
    * 고정된 길이 $k$ 안에서 최댓값/최솟값을 찾을 때.



---

## 3. 결정적 차이점 요약표

| 구분 | 투 포인터 (Two Pointers) | 슬라이딩 윈도우 (Sliding Window) |
| :--- | :--- | :--- |
| **포인터 이동 방향** | 서로 반대 방향 혹은 같은 방향 | **무조건 같은 방향** (시작/끝 모두 오른쪽) |
| **구간의 길이** | 대부분 **가변적**이며 불규칙함 | **고정**되거나 조건에 따라 확장/축소됨 |
| **핵심 키워드** | 쌍(Pair), 정렬(Sorted), 교환(Swap) | **연속된(Consecutive)**, 부분 구간(Subsegment) |
| **주요 연산** | 두 값의 비교 및 위치 변경 | 윈도우 내부 값의 **누적 합/빈도 갱신** |

---

## 4. 실전 전략 가이드
1. **배열이 정렬되어 있는가?** → 투 포인터 가능성이 높음.
2. **"가장 긴/짧은 연속된 부분 ~"을 묻는가?** → 가변 슬라이딩 윈도우.
3. **"길이가 $K$인 구간의 ~"을 묻는가?** → 고정 슬라이딩 윈도우.
4. **메모리 손상 없이 값을 수정해야 하는가?** → 투 포인터(In-place).
