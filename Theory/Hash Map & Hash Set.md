
# 🔑 Hash Map & Hash Set

데이터를 다루는 알고리즘 중 가장 효율적인 도구로, 특정 데이터를 찾거나 저장하는 시간을  **$O(n)$** 에서  **$O(1)$** 로 단축해 주는 강력한 자료구조입니다.

---

## 1. Hash, Hashing, Hash Function

단순히 데이터를 저장하는 것이 아니라, **"수학적 계산을 통해 데이터의 주소를 결정"** 하는 것이 핵심입니다.

* **Hash:** 임의의 데이터를 고정된 길이의 고유한 숫자 값으로 변환한 결과물입니다.
* **Hashing:** 데이터를 해시 값으로 변환하는 과정 그 자체를 의미합니다.
* **Hash Function:** 해싱을 수행하는 알고리즘입니다. 
    * 같은 입력값에는 항상 같은 결과(숫자)를 내놓아야 합니다.
    * 입력값이 아주 조금만 달라도 결과값이 완전히 달라져야 데이터가 골고루 분산됩니다.
    
---

## 2. Hash Map (Dictionary)과의 차이점

| 구분 | Hash Set | Hash Map (Dictionary) |
| :--- | :--- | :--- |
| **구조** | 값(Value)만 저장 | 키(Key)와 값(Value)의 쌍 |
| **중복** | 중복 허용 안 함 (자동 제거) | Key는 중복 불가, Value는 가능 |
| **주 용도** | 존재 여부 확인, 중복 제거 | 데이터 연결 및 빠른 검색 |

---

## 3. Hash Set: "값만 있는데 어떻게 찾을까?"

Hash Set은 내부적으로 **"값이 곧 키(Key)인 Hash Map"** 처럼 동작합니다. 별도의 인덱스나 키를 저장하지 않고도 $O(1)$ 탐색이 가능한 이유는 다음과 같습니다.

1.  **데이터가 주소가 됨:** `Set`에 `10`을 저장할 때, 시스템은 `hash(10)`을 계산합니다.
2.  **직접 접근:** 계산된 결과가 `5`라면, 배열의 5번 칸에 `10`을 넣습니다.
3.  **검색 원리:** "10이 있니?"라고 물으면, 다시 `hash(10)`을 계산해 5번 칸만 확인합니다. 다른 칸은 쳐다볼 필요도 없으므로 데이터 양에 상관없이 속도가 일정합니다.
4.  **결론:** 저장하려는 데이터 자체가 자신의 저장 위치를 결정하는 **키 역할**을 겸하기 때문에 별도의 키가 필요 없습니다.

---

## 4. 기술적 허점: 해시 충돌 (Hash Collision)

서로 다른 입력값이 해시 함수를 거쳤는데 하필 **똑같은 해시 값(주소)** 이 나오는 상황을 말합니다. 이는 수학적으로 피할 수 없으며, 이를 해결하는 방식은 크게 두 가지입니다.

### A. Chaining
같은 주소에 데이터가 겹치면, 해당 칸을 **연결 리스트(Linked List)** 구조로 만들어 데이터를 줄줄이 매답니다.
* **장점:** 구현이 단순하고 저장 공간 제약이 적습니다.
* **단점:** 한 칸에 너무 많은 데이터가 몰리면 탐색 속도가 $O(n)$까지 느려질 수 있습니다.

### B. Open Addressing
충돌이 발생하면 배열 내의 **다른 빈 칸**을 찾아 나섭니다.
* **Linear Probing (선형 조사):** 바로 다음 칸(`index + 1`)이 비었는지 확인하며 순차적으로 이동합니다.
* **Quadratic Probing (이차 조사):** 보폭을 제곱(`1, 4, 9...`)으로 늘려가며 빈 공간을 찾습니다.
* **Double Hashing (이중 해싱):** 별도의 두 번째 해시 함수를 써서 이동할 보폭을 결정합니다. 군집 현상을 피하는 가장 정교한 방법입니다.

---

## 5. 실전 적용 가이드 (Swift 기준)

1.  **Hashable 프로토콜:** Swift에서 `Dictionary`의 Key나 `Set`의 요소로 쓰려면 반드시 `Hashable`을 준수해야 주소 계산이 가능합니다.
2.  **순서 무관:** 해시는 주소를 계산해 무작위 위치에 저장하므로, **입력 순서가 보장되지 않습니다.**
3.  **용도:** "이전에 본 적 있는가?", "중복된 데이터인가?", "특정 키에 대응하는 값이 무엇인가?"를 풀 때 최우선으로 고려해야 합니다.
